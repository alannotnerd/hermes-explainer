Architecture

How Hermes Agent Works

A visual guide to the architecture

An open-source AI agent framework by Nous Research that runs on your terminal and connects to 10+ messaging platforms — one brain, everywhere.

Layer 1

The Gateway Layer

The gateway is a background service that connects to 10+ messaging platforms. All messages flow into one agent brain.

💬 Telegram 🎮 Discord 💼 Slack 🟢 WeChat 🔐 Signal 📧 Email 📱 SMS 🌐 Matrix 📲 WhatsApp 🍎 iMessage
⚙️ Hermes Gateway
Normalizes messages, manages platform-specific quirks, handles attachments, and queues everything into a single inbox for the agent.
Always on · systemd service
Layer 2 · The Core

The Agent Loop

Every message goes through this cycle. The agent can call tools multiple times before replying — it thinks, acts, observes, and repeats until it's ready.

STEP 1
Build Prompt
Assembles the system prompt from personality, loaded skills, and relevant memories
Personality Memory Skills
STEP 2
Call LLM
Sends the prompt to any provider — DeepSeek, Claude, GPT, Gemini, local models, or custom APIs
Tool call needed?
✓ YES ✗ NO
⚡ Execute Tool
Browser, Terminal, File I/O, Web Search, Memory, Delegation…
💬 Reply to User
result fed back
loop repeats
💡 The agent can loop multiple times per message. It calls a tool, reads the result, calls another tool, and only replies when it has everything it needs. This is what makes it an agent, not just a chatbot.

Layer 4

Skills & Memory — The Learning System

Hermes gets smarter with every conversation. Corrections become skills, facts become memories, and both load automatically next time.

SESSION 1
You correct Hermes
"Actually, when you deploy, always run docker compose up -d first." Hermes saves this as a SKILL.
SESSION 2
Skill auto-loads
You ask about deployment again. The skill loads automatically — Hermes already knows your procedure. No correction needed.
SESSION 3
Memory persists preferences
Hermes remembers you prefer Pacific Time, use pnpm over npm, and deploy to AWS us-west-2. It adapts without being told again.
📋
SKILLS
Procedural knowledge — how to do things. Reusable across sessions. Like macros on steroids.
💾
MEMORY
Factual knowledge — who you are, your preferences, past decisions. Grows with every interaction.
